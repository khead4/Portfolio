Voice Recognition — Accessibility Stress Test

Accessibility diagnosis

Voice is only helpful when the repair cost stays low.

This case is strongest when it reads as a failure analysis of trust, not as a generic speech-tech exercise. The real question was whether the interaction reduced effort or simply moved the effort downstream into verification, correction, and hesitation.

What I am proving

The work shows that accessibility depends on the full interaction path, not on the presence of a feature alone.

Why it matters

When a supposedly assistive input creates monitoring and repair work, the experience becomes less accessible even if the interface appears progressive.

What this demonstrates
  • UX writing as confirmation and guidance
  • Strategy as failure-pattern diagnosis
  • Product judgment around trust and recovery
Recorded break
"I'm gonna take a wok from the Chinese restaurant."

The result is close enough to pass a quick glance and wrong enough to distort meaning. That is a dangerous failure state because it pushes the user into constant checking rather than allowing the interaction to feel assistive.

What this exposes

Failure lives in the handoff.

The recognition engine matters, but the real usability problem sits between output, confirmation, and correction. That handoff decides whether the feature feels empowering or fragile.

Observed output

Target: “I’m gonna take a walk from the Chinese restaurant.”

Heard: “I’m gonna take a wok from the Chinese restaurant.”

Find
Discovery has to be obvious.

The user first needs a clear signal that voice exists, where it lives, and what kind of input it expects.

Speak
Natural phrasing should survive.

The input model needs to accommodate real language without turning every sentence into a formatting gamble.

Verify
Confidence should be visible.

Confirmation language should signal what was captured and what might still need attention without forcing a full audit.

Repair
Recovery should preserve momentum.

If something drifts, the user should be able to repair it quickly instead of being pushed back into a slower manual task.

Reading this case: the useful insight is not that voice can mishear. It is that failure cost emerges from the entire path around the recognition event.
UX writing implication

Confirmation language has to carry uncertainty without sounding alarmed.

The best recovery copy does not simply repeat the output. It tells the user what was captured, where confidence drops, and what can be corrected next without sending them back to the start of the interaction.

Product strategy implication

Recovery design decides whether the feature keeps its promise.

A feature is not judged only at the moment it works. It is judged by how gracefully it protects the user's momentum once confidence starts to slip.

  • Surface confidence instead of hiding it inside a reread burden.
  • Preserve momentum with light-touch correction choices.
  • Treat repair as part of accessibility, not as an afterthought.
Portfolio value: this is a trust-and-repair case, not a speech-tech summary. The work turns a narrow test into a clearer argument about product writing, confidence, and the hidden cost of correction.

Screen Reader Structure Analysis

Page 1

Screen Reader Structure Analysis

This study documents what happened when the New York Public Library site was explored through screen reader output alone. Instead of relying on visual hierarchy, the analysis reconstructs the page from headings, labels, and link announcements, making the site's structure visible through absence, repetition, and ambiguity.

The exercise revealed how much navigation depends on semantic clarity. When headings were sparse, labels were vague, or alt text was missing, the page no longer functioned as a coherent system. The result was not simply slower movement, but a different experience of the interface altogether - one based on inference rather than guidance.

Core focus
  • Heading logic and content grouping
  • Link clarity and navigation predictability
  • The effect of missing or vague alternative text
  • How audio-only navigation reshapes the mental model of the page
Visual reading
A sighted user receives grouping, scale, and emphasis all at once, so structure often feels obvious before the details are even processed.
Audio reading
A screen reader user receives the page as a sequence. Weak headings, vague labels, and absent alt text therefore become structural failures, not cosmetic ones.
Design rule
Semantic structure is part of the interface itself. If it breaks in audio, the design logic was not coherent enough to begin with.
Page 2

Key Findings and Reflection

Findings

The site could still be traversed, but not with confidence. The structure depended too heavily on visual interpretation, which meant the screen reader user had to assemble the page manually from fragments. Navigation cues that are visually obvious did not consistently translate into meaningful spoken guidance.

Interpretation

This made the interface feel less like a designed path and more like a reconstruction task. Missing labels, weak headings, and limited descriptive support forced the user to infer relationships between content blocks that should have been directly communicated by the page itself.

Overall Reflection

The analysis shows that accessibility is not only about adding tools, but about preserving structure across modes of perception. When semantic hierarchy is weak, the experience fractures. A screen reader does not merely reveal missing accessibility support - it reveals whether the underlying design logic is actually coherent.

Structured page

Meaning stays attached

Strong headings, clear labels, and useful alt text preserve the relationships between content blocks so the user can move with confidence.

Weak page

The user has to reconstruct it

When semantic cues are missing, the page stops feeling authored and starts feeling like fragments the user has to assemble manually.

3 Coffee Shops, 3 Cities — Website Heuristic Analysis

Heart Roasters Homepage

The homepage of Heart Roasters demonstrates a strong emphasis on Aesthetic and Minimalist Design (Heuristic #8). The layout is clean, visually appealing, and uncluttered, featuring a rotating set of large images that highlight featured coffees, merchandise, and subscription offerings. Because only one primary message is shown at a time, the design avoids overwhelming users with excessive information. Optional sections such as best sellers and location hours appear lower on the page, allowing users to explore additional content without forcing it upon them.

The site also supports Recognition Rather Than Recall (Heuristic #6). Product images, packaging, and promotional visuals make it easy for users to identify offerings without needing to remember names or details. For example, the image of the subscription box containing two coffee bags communicates the concept of a delivery service immediately. This visual approach reduces cognitive effort and supports quick decision-making.

Heart Roasters — Feedback and Carousel Visibility

However, the rotating carousel weakens Visibility of System Status (Heuristic #1). Users may not realize that the images are part of a slideshow or understand how many slides exist. While navigation dots are present, they are small and visually subtle, which may reduce discoverability. Without clear feedback or controls, users may miss important content if the slide changes automatically.

Heart Roasters — Navigation and Typography

The navigation menu demonstrates Consistency and Standards (Heuristic #4) by using familiar e-commerce categories such as Shop, Locations, Learn, and Wholesale. These labels align with user expectations and industry conventions, making the site easier to understand. The subcategories within the shop menu are logically organized, supporting efficient navigation.

At the same time, some usability issues arise from typography choices. Call-to-action text such as "SUBSCRIBEANDSAVE"lacks spacing between words, which reduces readability and slows scanning. This issue affects both Match Between the System and the Real World (Heuristic #2) and Recognition Rather Than Recall (Heuristic #6) because users must spend extra effort interpreting the text rather than understanding it instantly.

Overall, the design achieves an effective balance between aesthetics and usability. The minimalist layout enhances visual appeal and prevents cognitive overload, while recognizable imagery and standard navigation support task completion. Minor adjustments to feedback, typography, and carousel controls would further improve usability without compromising the site's clean visual identity.

3 Cities, 3 Coffees — Insights

Strategic Insight

Atmosphere only works when orientation survives it

The clearest pattern is that strong visual identity does not automatically produce strong usability. All three sites communicate mood effectively, but they do not all preserve feedback, legibility, or continuity with the same discipline.

That makes the real lesson strategic: brand expression is only an advantage when it helps the user understand where they are, what they can do, and how the system will respond next.

UX Interview Critique & Analysis Activity — Critique Activity

Question Design

The quality of the interview is decided before the interview starts.

A biased question does not only sound awkward — it changes what kind of evidence becomes possible to collect at all. The move is to identify the bias pattern, understand what it blocks, and rewrite toward neutral, present-tense, behavior-based prompts.

Sentence surgery — Q1

The flaw becomes obvious when original and repair sit beside each other

Original prompt

Leading and emotionally framed

"Do you find it frustrating when professors assign too much work?"

  • Presumes the workload is excessive before the participant can define it.
  • Cues frustration as the expected emotional register.
  • Agreement becomes easier than contradiction or nuance.
Strip assumption
Rewritten prompt

Neutral and experience-first

"Can you describe your experience with the workload across your classes?"

  • The participant decides whether the workload is balanced, excessive, or manageable.
  • Description before evaluation protects the evidence.
  • The answer can now include comparison, routine, tradeoff, or contradiction.
Rewrite rule: remove the interviewer's emotion, judgment, and built-in conclusion. The better question creates more room for the participant to tell you what is actually true.
Editorial read: strong interview questions are not just cleaner sentences. They are better evidence structures.
Questions 1 – 2

Leading & Assumptive Questions

Scenario: Understanding how graduate students manage academic deadlines  ·  Author: Kaitlyn Head

1
Leading
"Do you find it frustrating when professors assign too much work?"
⚠ What's wrong

Assumes workload is excessive and frames the experience emotionally. Pressures the participant to agree even if their reality differs. Bias: leading + assumptive.

✓ Stronger version

"Can you describe your experience with the workload across your classes?"

2
Assumptive
"How often do you procrastinate?"
⚠ What's wrong

Assumes the participant procrastinates, risking defensiveness. Focuses on frequency without understanding behavior or context. Bias: assumptive + closed.

✓ Stronger version

"How do you typically manage your time when completing assignments?"

Questions 3 – 5

Hypothetical, Closed & Judgmental Questions

3
Hypothetical
"Would you use an app that helps organize your deadlines?"
⚠ What's wrong

Asks about imagined future behavior — participants often say yes to hypotheticals regardless of actual intent. Skips understanding current tools and real friction. Bias: hypothetical + solution-focused.

✓ Stronger version

"How do you currently keep track of your deadlines?"

4
Closed
"Is time management hard for you?"
⚠ What's wrong

Closed yes/no that assumes difficulty. Invites defensiveness and produces no usable description of strategies or experience. Bias: closed + leading.

✓ Stronger version

"Can you tell me about your experience managing your time and any approaches you've found helpful?"

5
Judgmental
"Why don't students plan ahead more?"
⚠ What's wrong

Frames students as universally poor planners. Implied blame produces defensive or performative answers rather than honest reflection. Bias: judgmental + assumptive.

✓ Stronger version

"What factors have affected your studies, and how have you responded to them?"

Repair pattern

Bad interview questions reveal their bias quickly

Question shape

Hypothetical or closed

Asks for polished prediction or yes/no instead of grounded description of real behavior.

What it causes

Shallow and defensive answers

The participant has less room to describe routines, workarounds, frustrations, or context.

Better move

Ask from lived practice

Present-tense questions about what the person does now open up richer stories and clearer evidence.

Questions 6 – 8

Agreement, Stress & Solution Bias

6
Leading
"You prefer digital calendars over paper planners, right?"
⚠ What's wrong

The phrase "right?" is a confirmation cue — it pulls the participant toward agreeing with the interviewer's expectation rather than expressing their actual preference. Bias: leading + agreement.

✓ Stronger version

"Do you use digital calendars, paper planners, or another method? What do you like about it?"

7
Hypothetical
"Would reminders reduce your stress?"
⚠ What's wrong

Hypothetical and double-assumptive — presumes both that reminders help and that the participant experiences stress. Invites only a yes/no. Bias: hypothetical + assumptive.

✓ Stronger version

"Do you use reminders to manage your tasks? If so, how do they affect you?"

8
Solution Bias
"What features should a better deadline system include?"
⚠ What's wrong

Asks the participant to design before the study understands their current system. Assumes the existing system is insufficient and skips all understanding of real behavior. Bias: solution-focused + assumptive.

✓ Stronger version

"How do you currently keep track of deadlines?"

Interview discipline

Keep the conversation in the present, not in imagined solutions

Agreement bias: phrases like "right?" pull the participant toward confirming the interviewer's expectation. Solution bias: feature questions ask the participant to design before the study understands the current system. Stronger route: ask what they use now, what works, and where friction keeps showing up.
What to remove

Confirmation cues, leading assumptions, and premature design language.

What to ask instead

Neutral follow-ups about current tools, behaviors, tradeoffs, and unmet needs.

Why it matters

The better question is the one that produces usable insight, not agreement.

Interview Reading

A comfortable conversation can still collect distorted evidence.

This reflection reads as a critique of interviewer behavior — not just a recap of what the participant said. The value is in identifying where confirmation, premature evaluation, and missed probing began shaping the data before it could speak for itself.

Analysis — Page 1

Where the interview went wrong

Conducting an interview with a peer provided valuable insight into both the participant's study habits and my own interviewing skills. While the conversation flowed comfortably, reviewing the transcript revealed areas where bias appeared, opportunities for deeper probing were missed, and improvements could be made for future interviews.

Bias appeared primarily in the form of leading or assumptive language. This question assumed that the digital calendar was the primary method, potentially limiting discussion of alternative strategies. Similarly, statements such as "That sounds very helpful" or "It seems like you do a good job" introduced positive evaluation, which may have influenced the participant to continue emphasizing successful strategies rather than discussing challenges or failures. These responses reflect confirmation bias, as I tended to validate information that aligned with the topic of time management tools instead of remaining neutral.

One missed probing opportunity occurred when the participant mentioned that deadlines used to "creep up" on them and cause lost sleep before they adopted the digital calendar. This was a moment where I could have explored the emotional, academic, or behavioral impact in more depth. Follow-up questions such as "How did that affect your academic performance?" or "What prompted you to change your system at that time?" could have provided richer data about motivation and consequences. Instead, I acknowledged the stress but moved on to the next prepared question, which limited the depth of insight gained.

Botanic Elixir — Strategy to Site Decisions

Strategy to Site

Every site decision came from a strategic brief. This is the chain.

The business plan answered why Botanic Elixir should exist. This page answers something different: which specific decisions shaped the site, and what in the brief made each one necessary. The five decisions below are not design preferences — each one was forced by a strategic condition that had already been written.

The brief established four things that the site had to solve: the concept was unfamiliar, the audience arrives in a specific state rather than with a specific goal, the offer had to be legible enough to choose on a first visit, and the experience had to be coherent enough to return to. Every structural and copy decision on the site follows from one of those four conditions.

Five Decisions — Brief to Build

What the brief required and what the site did about it

1
Information Architecture Forced by: state-based arrival
Structure navigation around arriving states, not product categories
Brief condition

The audience does not arrive with a product goal. They arrive carrying a mood or a need — focus, pause, connection, recovery. A product-first IA would require them to translate that need into a category before they could act. That is one extra cognitive step the concept cannot afford on a first visit.

High — affects entire site structure
Site decision

Navigation organised around states: Focus, Pause, Connect, Restore. Each state routes to the relevant spatial zone, drink pairings, and seating context. The user identifies their need first; the site then tells them where to go.

"I need to focus" → Focus zone → quiet seating + single-origin brew pairings + no bookable events
2
Copy Tone Forced by: unfamiliar concept
Write to clarify the unfamiliar, not to heighten the atmospheric
Brief condition

A non-alcoholic adaptive café is genuinely unfamiliar. The risk of purely atmospheric copy is that it adds beauty while the reader still does not know what the place is, what they would drink, or whether it is for them. The brief required copy that earns comprehension before it earns admiration.

High — sets voice for all copy
Site decision

Every section opens with a functional statement before any atmospheric language. Headers name the offer plainly; supporting copy adds texture. The hierarchy is: what this is → what you do here → why it is worth it.

"Botanical drinks crafted for your current state. No alcohol. No compromise on intention." — not "A sensory journey into botanical harmony."
3
Navigation Labels Forced by: first-visit legibility
Use plain verbs in navigation, not brand vocabulary
Brief condition

Brand language that replaces functional labels creates a decoding cost. "The Apothecary" and "The Ritual Room" communicate personality but not function. On a first visit, the user needs to know where to go before they can appreciate where they are.

Medium — affects entry comprehension
Site decision

Navigation uses plain verbs and nouns: Drinks, Visit, Book, Learn. Brand names appear as subheads within sections once the user has already located themselves. The label does the orienting; the brand name does the colouring.

Nav: "Drinks" → Section title: "The Apothecary Menu" — one clarifies, the other distinguishes
4
Visual Restraint Forced by: atmospheric concept
Keep the visual system quiet so the concept can speak
Brief condition

The concept is atmospheric by nature. A visually loud site competes with the atmosphere it is trying to communicate. The brief required restraint as a design discipline — the visual system should support the experience without performing it.

Medium — shapes visual identity
Site decision

Dark backgrounds, one accent colour, generous white space, and a single serif/sans pairing. No hero carousels, no animated backgrounds, no parallax. Motion only where it earns meaning — the timed letter-reveal on the CTA is the only active element on load.

Visual rule: if removing an element makes the page feel clearer, remove it.
5
Return Logic Forced by: repeat business model
Build the site so a second visit feels different from the first
Brief condition

The brief identified return as non-negotiable for the business model. A concept-driven café cannot survive on novelty visits. The site had to give people a reason to come back — which meant the experience had to model the café itself: different states, different routes, different reasons.

High — determines long-term value
Site decision

The state-routing structure means the site presents differently depending on which entry point a user takes. Someone arriving for focus sees a different primary content path than someone arriving for connection. The structure supports re-entry without sameness.

No single "home page experience" — each state pathway is a distinct entry into the same space.
Strategic read: the site is not a design exercise built on top of a business plan. Every structural choice was constrained by what the brief required the site to do. The five decisions above are where that constraint became visible.

Page 2 — Original Micro-Interaction Prototype

The “Enter” reveal — a four-state sequence

The original design centres on the label Enter appearing letter by letter after page load. The gradual reveal acts as a timed feedback mechanism: it signals that the interface is active, draws attention toward the call-to-action, and creates anticipation without adding visual noise or requiring prior input.

1
Stillness
Trigger: page load
What happens

The page arrives completely quiet. No animation, no motion, no sound. The button exists but its label does not yet.

Why it matters

Stillness is not emptiness — it is the condition that makes the reveal meaningful. Without a quiet frame, the motion that follows has no contrast to work against.

2
Reveal
Pacing: 1 letter / second
What happens

Letters appear one at a time — E, then N, then T, then E, then R — at one-second intervals, creating a controlled typing-style rhythm.

Why it matters

Loading time becomes visible invitation rather than empty delay. The user is not waiting — they are watching a process resolve. That shift reduces perceived friction without changing actual load speed.

3
Attention shift
Effect: eye movement
What happens

The user's eye is pulled toward the button as the word builds. The motion creates a gentle visual pull without requiring a large gesture, bright colour, or animation loop.

Why it matters

Directing attention through timing rather than scale or brightness keeps the rest of the page calm. The interface stays restrained while still guiding the user toward the intended action.

4
Ready to act
Outcome: committed state
What happens

The word "Enter" is complete. The button is fully legible. The interface is now in its active state — prepared for the user's input.

Why it matters

The sequence communicates system readiness without a loading spinner, status message, or prompt. The interaction itself is the feedback — motion used as signal, not as decoration.

Page 2 — System Anatomy

Four layers — rule, application, and failure mode

The style guide becomes most useful when it is read as a brand system rather than a static board. These four layers are the parts that must stay coherent as new interfaces and materials are designed. Each has a governing rule, a clear application, and a failure mode that becomes visible when the rule is ignored.

1
Typography — Voice through hierarchy
Space Grotesk · Source Serif 4 · Space Mono
System rule

Serif display moments create atmosphere and editorial weight. The supporting sans system keeps interface language direct and legible. Mono is reserved for system labels and technical annotation — never used for body copy.

Failure if ignored

Beautiful display moments appear but the reading rhythm becomes inconsistent. Users must re-calibrate between sections, which erodes trust in the system even if each individual choice looks considered.

2
Colour — Atmosphere with control
Onyx · Copper · Rose Gold · Graphite
System rule

The four-colour palette carries emotional tone while contrast discipline keeps the brand usable across dark and light surfaces. Copper is the emphasis colour — it should appear where the system wants the user's attention, not as ambient decoration.

Failure if ignored

Pages drift into unrelated moods and stop reading as one brand world. When Copper appears everywhere, it ceases to direct attention and becomes background noise instead of a signal.

3
Components — Repeatable structure
Cards · Buttons · Rails · Boards
System rule

Cards, buttons, panels, and section shells share one structural grammar. Padding, border-radius, and spacing ratios are fixed — new components are built from these primitives, not invented separately for each context.

Failure if ignored

Each new page looks like a one-off composition instead of part of a coordinated system. The work feels assembled rather than authored — technically competent but lacking the coherence that signals a real visual identity.

4
Imagery — Visuals that reinforce meaning
Network · Constellation · Abstract structure
System rule

Photography and graphics should help interpret the concept, not decorate it or compete with it. The preferred visual direction favours network and constellation structures over literal productivity imagery — abstract but purposeful.

Failure if ignored

The work feels either too literal (stock photos of laptops) or too decorative (purely aesthetic textures), and the concept loses precision. Imagery becomes atmosphere without argument.

Guide purpose: define a reusable visual system that stays legible, coherent, and adaptable as the project grows — not a mood board that describes one moment.
